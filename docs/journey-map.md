# Rehearse — User Journey Map

**Persona:** Alex, Senior Product Designer, actively interviewing for an AI Product Design role. Strong portfolio, uncertain about AI-specific interview norms. Real interviews in 5 days. High stakes, high anxiety.

**Scenario:** First-time use of Rehearse, preparing for a Hiring Manager round.

**Journey boundaries:** Pre-discovery anxiety → Post-session, activated to act on feedback.

---

## Stage 1 — Arrival
*"Is this going to be worth my time?"*

| | |
|---|---|
| **Goal** | Quickly determine if this tool is different from every other generic prep resource |
| **Actions** | Lands on home, reads value prop, scans for trust signals, decides to start |
| **Touchpoints** | Home screen, headline, CTA |
| **Thoughts** | *"Every tool says it's personalized."* / *"What makes this different from Googling?"* / *"No login — low commitment, okay."* |
| **Emotion** | 😐 Cautious — **3/10** |
| **Pain points** | Generic AI product language actively erodes trust with this user. Vague aspirational copy reads as dishonest. |
| **Opportunities** | Headline must be specific and direct. Concrete specificity ("researches the company, simulates the interview, tells you precisely where you stood") lands harder than "AI-powered personalized prep." |

---

## Stage 2 — Context Intake
*"Okay, let's see if it actually uses any of this."*

| | |
|---|---|
| **Goal** | Get through setup without feeling like it's just collecting data for no reason |
| **Actions** | Uploads resume, pastes portfolio URL, pastes job posting URL, selects stage |
| **Touchpoints** | File upload, URL inputs, stage selector |
| **Thoughts** | *"Will it actually read my portfolio or just pretend to?"* / *"What changes based on which stage I pick?"* |
| **Emotion** | 🙂 Engaged but uncertain — **5/10** |
| **Pain points** | No feedback confirming inputs were understood. Stage selector feels abstract without knowing what changes. |
| **Opportunities** | Brief inline explanation of what each input unlocks. Stage descriptions that show what changes. Parsing confirmation ("Resume read — 3 years at Company X") builds immediate trust. |

---

## Stage 3 — Company Research
*"This is the moment I find out if it's real."*

| | |
|---|---|
| **Goal** | Get genuinely useful, company-specific intelligence — not a surface-level summary anyone could Google |
| **Actions** | Watches research load with live status, reads coverage indicator, reviews company brief, decides to proceed |
| **Touchpoints** | Research loading screen, coverage indicator, company brief |
| **Thoughts** | *"Is it actually searching or faking a loading state?"* / *"I've seen almost nothing about this company online."* / *"If it says 'high coverage' but gives generic questions, I'll know it lied."* |
| **Emotion** | 😬 High stakes — **4/10** (rises to **7/10** if data is real and specific) |
| **Pain points** | This is the highest-risk moment for trust. Confident-sounding but vague results permanently damage credibility. Sparse data for smaller companies needs honest acknowledgment. |
| **Opportunities** | **Moment of truth #1.** Show sources in real time. Be explicit when data is thin: "We found limited information — your questions will be well-informed but less company-specific." Honesty here is more trust-building than false certainty. |

---

## Stage 4 — Interview Simulation
*"Oh god, I have to actually say this out loud."*

| | |
|---|---|
| **Goal** | Experience something that feels like a real interview, not a quiz |
| **Actions** | Reads persona intro, activates mic, speaks answer, reviews transcript, submits, repeats × 7 |
| **Touchpoints** | Persona intro, question display, mic button, live transcript, submit |
| **Thoughts** | *"This is more uncomfortable than I expected."* / *"I sound terrible."* / *"I keep saying 'like' and 'um'."* / *"Is this persona going to be weirdly nice to me?"* |
| **Emotion** | 😰 Vulnerable — **2/10** early → **5/10** by question 4 as rhythm settles |
| **Pain points** | Voice input is deeply uncomfortable. Seeing raw transcript is another layer of exposure. Instinct is to re-record endlessly. Persona going out of character breaks immersion. |
| **Opportunities** | **Moment of truth #2.** Persona must feel real — follow-up questions, not just queued prompts. Transcript editing: one pass, low friction, then move on. Don't show scores mid-session — users perform for the score instead of practicing. |

---

## Stage 5 — Feedback Report
*"Am I going to find out I was worse than I thought?"*

| | |
|---|---|
| **Goal** | Understand exactly where they were strong and where they failed — with enough specificity to fix it |
| **Actions** | Reviews report, reads per-question scores, reads what worked / what to improve, reads sample stronger answers, decides what to do next |
| **Touchpoints** | Feedback report, score display, sample answers, exit options |
| **Thoughts** | *"I knew that answer was weak."* / *"I didn't expect that one to score low — what did I miss?"* / *"The sample answer is good but doesn't sound like me."* / *"This is actually useful."* |
| **Emotion** | 😣 Defensive → 😤 Motivated — **3/10** → **7/10** |
| **Pain points** | Generic feedback ("be more specific") is useless. Sample answers in a different voice are useless. Score without context raises anxiety without direction. |
| **Opportunities** | **Moment of truth #3.** Frame feedback as a solvable gap: "You described the outcome but didn't connect it to the business impact — that's the missing piece." Sample answers should calibrate instincts, not script responses. |

---

## Stage 6 — Post-Session Decision
*"What do I do now?"*

| | |
|---|---|
| **Goal** | Leave with a clear sense of what to work on and confidence the time was well spent |
| **Actions** | Copies report, chooses redo / new stage / start fresh / exit |
| **Touchpoints** | Three exit CTAs, copy report action, session-end notice |
| **Thoughts** | *"I should redo the AI fluency questions specifically."* / *"I need the recruiter screen too."* / *"I want to save this."* |
| **Emotion** | 💪 Activated — **7/10** (if feedback was honest and useful) |
| **Pain points** | No way to save in V1 without copy action. Equal-weight exit options don't guide toward the right next step. |
| **Opportunities** | Exit options need brief context ("Redo this stage — sharpen the answers you just gave"). Low-friction report copy (plain text, no auth) removes post-session anxiety at near-zero cost. |

## Moments of Truth

| # | Moment | What's at stake |
|---|---|---|
| 1 | First 10 seconds on home | Trust or bounce. Generic copy = gone. |
| 2 | Company research results | Credibility or collapse. Thin data acknowledged honestly = trust. Thin data papered over = poisoned session. |
| 3 | First simulation question | Immersion or detachment. Real persona = lean in. Chatbot feel = go through the motions. |
| 4 | First piece of critical feedback | Useful honesty or demoralizing noise. Determines whether they come back. |

---

## Top 5 Design Opportunities

| Priority | Opportunity | Why it matters |
|---|---|---|
| 1 | Transparent company research with live source attribution | Builds the trust that makes everything downstream credible |
| 2 | Persona realism — follow-up questions, not just queued prompts | Turns simulation from a quiz into an actual practice rep |
| 3 | Feedback framing: gap + fix, not just score | Changes the emotional experience from judgment to coaching |
| 4 | Transcript editing: one pass, low friction | Reduces perfectionism spiral that disrupts authentic practice |
| 5 | Session-end report copy / no auth required | Removes significant post-session anxiety at near-zero implementation cost |
